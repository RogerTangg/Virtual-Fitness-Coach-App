# 檔案：技術選型與架構 (v1.0) - 虛擬健身教練

## 1. 總覽

本文件旨在說明「雲端虛擬健身教練互動應用」MVP 版本所採用的技術棧，並闡述每一項選擇的理由。

所有決策都圍繞著核心需求：**大學生團隊、低開發門檻、零金錢成本、易於維護，並使用 Render 部署。**

---

## 2. 總體架構：Jamstack + BaaS

我們將採用一個**解耦 (Decoupled)** 的架構。前端（你的 React 應用）和後端（你的 Supabase 數據）是完全分離的。

* **前端 (Frontend)：** 一個靜態網站 (Static Site)，負責所有 UI 介面和業務邏輯。
* **後端 (Backend)：** 一個「後端即服務」(BaaS) 平台，僅負責提供 CMS 功能和數據 API。

這種架構是現代網頁開發的主流，它能提供最佳的性能、安全性和最低的維護成本。

---

## 3. 詳細技術選型

### 3.1. 前端 (Frontend)

* **框架 (Framework)：** **React**
* **建構工具 (Build Tool)：** **Vite**
* **說明：**
    * **為什麼選擇 React？**
        1.  **網頁優先 (Web First)：** React 是建構現代、高效能網頁應用的業界標準。
        2.  **為跨平台鋪路 (Cross-Platform)：** 這是最重要的理由。你們團隊未來若要支援行動裝置，從 React 遷移到 **React Native** 是最無痛、最快的路徑。你們甚至可以在 Web 和 Mobile 之間共享大量的程式邏輯 (例如「課表生成器」演算法)，極大降低未來的開發成本。
        3.  **龐大的生態系：** 任何你們需要的功能（計時器、語音提示），React 社群都有現成的解決方案。
    * **為什麼選擇 Vite？**
        1.  **極速開發體驗：** Vite 提供了近乎即時的開發伺服器啟動速度和熱更新 (HMR)，這會讓你們的開發和除錯過程非常愉快。

### 3.2. 後端、資料庫 & CMS

* **服務 (Service)：** **Supabase**
* **說明：**
    * **為什麼選擇 Supabase？ (而不是自己架設後端)**
        1.  **完美符合「易於管理」：** 根據 PRD (Epic 1)，你們需要一個 CMS 來管理「運動數據資料庫」。Supabase **就是**你們的後端、資料庫、和 CMS **三合一**的解決方案。
        2.  **零後端程式碼：** Supabase 提供了一個託管的 PostgreSQL 資料庫，並**自動**為你的 `exercises` 資料表生成 RESTful API。你們團隊**不需要**撰寫、部署或維護任何後端 API 程式碼。
        3.  **免費且強大：** Supabase 的免費方案對於你們 MVP 專案的流量和數據量來說綽綽有餘。
        4.  **易用的 CMS 介面：** 你們可以像使用 Excel 一樣，在 Supabase 的網頁介面上新增、修改健身動作，完全符合「低技術門檻」的需求。

### 3.3. 部署 (Deployment)

* **平台 (Platform)：** **Render**
* **說明：**
    * **為什麼選擇 Render？**
        1.  **滿足團隊要求：** 這是你們指定的核心需求。
        2.  **完美的前端部署方案：** 我們將使用 Render 的 **"Static Site" (靜態網站)** 服務來部署 React 應用。
        3.  **成本為零：** Render 的靜態網站服務是**完全免費的**，沒有「休眠」(spin down) 問題，可確保學員 (Trainee) 隨時都能快速訪問。
        4.  **自動化 (CI/CD)：** 你們只需要將 Render 帳號連接到 GitHub 倉庫。未來只要你們推送 (push) 程式碼到 `main` 分支，Render 就會**自動**抓取新程式碼、重新建構 (build) 並部署你們的網站，極度方便。

---

## 4. 架構總結

這個「**React on Render + Supabase**」的組合，我稱之為「務實的 MVP 啟動包」。

| 元件 | 技術 | 部署平台 | 成本 | 維護工作 |
| :--- | :--- | :--- | :--- | :--- |
| **前端** | React (Vite) | **Render (Static Site)** | **免費** | **低** (只需寫程式) |
| **後端** | Supabase | Supabase Cloud | **免費** | **零** (服務由 Supabase 託管) |
| **資料庫** | PostgreSQL | Supabase Cloud | **免費** | **零** (服務由 Supabase 託管) |
| **CMS** | Supabase UI | Supabase Cloud | **免費** | **零** (服務由 Supabase 託管) |

這個架構讓你們（開發者）可以 100% 專注於打造前端 React 應用程式的功能 (Epic 2, 3, 4)，而將所有「後端雜事」(Epic 1 的伺服器、資料庫、API、CMS 維護) **完全外包**給 Supabase，並透過 Render 實現零成本的自動化部署。
